# DeepSeek Trading Bot — TR-GC-Crypto-9

Standalone Python automation script for Bot ID **25880**. A cron-schedulable pipeline using
**DeepSeek V4 Pro** for trading decisions, **Signum MCP** for data + order execution,
and **Telegram** for notifications.

## Architecture

```
READ ──────► DECIDE ──────► WRITE ──────► VERIFY ──────► NOTIFY
Signum MCP    DeepSeek       Signum MCP    Re-fetch       Telegram
(get-bot-     V4 Pro         (send-        HOLDINGS       (optional)
assets +      (structured    trading-
trend-radar)  JSON orders)   signal)
```

1. **READ** — Fetch HOLDINGS + Crypto Trend Radar ("gc" detector) via Signum MCP JSON-RPC
2. **DECIDE** — DeepSeek V4 Pro (reasoning model) evaluates strategy rules against current data, returns `{"orders": [...]}`
3. **WRITE** — Orders sent one-at-a-time via Signum MCP `send-trading-signal`; sizing recomputed fresh before each order; exits processed before entries
4. **VERIFY** — Re-fetch HOLDINGS, compare expected vs actual end-state
5. **NOTIFY** — Telegram summary: NAV before/after, orders sent/skipped, errors, auto-conversion warnings

## Quick Start

```bash
# Clone
git clone https://github.com/4mrm4/deepseek-signum-trading-bot.git
cd deepseek-signum-trading-bot

# Install dependencies (no mcp SDK required — uses direct JSON-RPC over HTTP)
pip install -r requirements.txt

# One-time OAuth setup (opens browser, registers client, obtains refresh token)
python setup_oauth.py

# Add your DeepSeek API key
echo "DEEPSEEK_API_KEY=sk-your-key-here" >> .env

# Optional: Telegram notifications
# Add TELEGRAM_BOT_TOKEN and TELEGRAM_CHAT_ID to .env

# Dry-run (default — logs trades, sends nothing to exchange)
python deepseek_trading_bot.py

# Live (sends real orders)
python deepseek_trading_bot.py --live
```

## Environment Variables

All secrets via `.env`:

| Variable | Required | Purpose |
|---|---|---|
| `DEEPSEEK_API_KEY` | Yes | DeepSeek V4 Pro API key (get from https://platform.deepseek.com) |
| `SIGNUM_CLIENT_ID` | Yes | OAuth client ID (auto-generated by `setup_oauth.py`) |
| `SIGNUM_REFRESH_TOKEN` | Yes | OAuth refresh token (auto-generated by `setup_oauth.py`) |
| `TELEGRAM_BOT_TOKEN` | No | Telegram Bot API token for notifications |
| `TELEGRAM_CHAT_ID` | No | Telegram chat ID for notifications |

Never commit `.env` — it's in `.gitignore`. `setup_oauth.py` runs once to populate the Signum tokens.

## Strategy (TR-GC-Crypto-9) — Verified against spec as of 2026-06-21

The bot implements the full TR-GC-Crypto-9 ruleset. DeepSeek V4 Pro makes the
entry/exit decisions using the strategy system prompt; Python enforces every
quantitative guardrail mechanically (not via LLM).

### Long Exits
Close **100%** of any held asset (excluding stablecoins/fiat) when:
- The latest closed candle's **close is below** the upper GC band, or
- The asset is **no longer present** in Trend Radar results

Proceeds go to **USDC**, **USDT**, or **USD** (priority order).

### Long Entries
Enter a long when:
- The latest closed candle's **close crossed above** the upper GC band
  (this candle close > GC upper **AND** previous candle close ≤ GC upper)
- Asset is **not already held** (one position per coin unless dust)
- Asset was **not exited** earlier in the same run
- Trend Radar sorted by `marketRank` ascending, first 50 rows

### Entry Sizing (computed from NAV snapshot, never recomputed mid-run)
| Breakout recency | Entry size |
|---|---|
| `breakoutDate` ≤ 25 calendar days ago | 8% of NAV |
| Older breakout or no breakout | 2% of NAV |

A breakout is NOT required to enter — it only sets the position size.

### orderSize Calculation (recomputed before EACH order)
```
target_usd  = X% of NAV (snapshot from step 1)
order_size% = (target_usd / current_quote_balance) × 100
capped at 100%
```
This prevents stale percentages from under-sizing later orders as quote balance shrinks.

### Safety Guardrails (enforced in Python only — never in the LLM prompt)

| Guard | Rule |
|---|---|
| **Min order value** | Entry: ≥ $10 AND ≥ 1% of NAV. Exit: ≥ $10 |
| **Ticker collision** | Skip if exchange price diverges >50% from reference (`cmcRefPriceUsd` → newest `ohlc.c` fallback). Skip if either is missing. |
| **Schema validation** | Abort run if DeepSeek returns invalid JSON or fails Pydantic `OrdersResponse` validation |
| **Dry-run default** | `--live` flag required to send real orders. Default logs only. |
| **Idempotency** | Unique microsecond timestamps per order prevent duplicate signals |
| **Same-asset re-entry** | Cannot buy an asset that was sold earlier in the same run |
| **Trend radar minimum** | ≥ 50 rows required; retry up to 3 times |
| **Auto-convert logging** | Logs `AUTO-CONVERT` warning when an entry's quote coin isn't held — assumes Signum auto-converts from available stablecoins. Verify with a small trade before `--live`. |

### Audit Trail

Every run saves the full DeepSeek prompt + response to `decisions/run-YYYYMMDDTHHMMSSZ.json`.
Review these files to understand the model's reasoning for each trading decision.

## Scheduling (cron-compatible, one-shot)

```bash
# Every 6 hours
0 */6 * * * cd /path/to/bot && python deepseek_trading_bot.py --live

# Daily after Trend Radar update
0 6 * * * cd /path/to/bot && python deepseek_trading_bot.py --live
```

Also works with Windows Task Scheduler, GitHub Actions, or any scheduler.
